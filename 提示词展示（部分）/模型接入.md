# 云端大模型 API 接入 — 概述

为「智能课程表 Pro」项目接入云端大模型 API，新建 `llm_client.py` 作为韧性调用层，
并改造 `app.py` 接入，满足全部六项需求。

## 交付文件
| 文件 | 说明 |
|------|------|
| `llm_client.py` | 核心：韧性大模型客户端（超时/重试/熔断/限流/日志/密钥管理） |
| `llm_config.json` | 配置：base_url / model / timeout / mock + retry / circuit_breaker / rate_limit / logging |
| `.env.example` | 密钥模板（复制为 `.env` 填入 `LLM_API_KEY`，不入库） |
| `.gitignore` | 忽略 `.env`、`__pycache__` |
| `app.py` | 已改造：用 `get_llm_client().chat()` 替换旧的 `call_llm` |

## 六项需求落地
1. **密钥管理** — `api_key` 仅从环境变量 / `.env` 读取（`LLM_API_KEY`），绝不硬编码；
   优先级：环境变量/.env > llm_config.json > 默认值。
2. **超时处理** — 每次请求独立 `timeout`（默认 30s），阻断长时间阻塞。
3. **自动重试** — `RetryConfig`：可配置 `max_attempts`、指数退避 + 抖动；仅对
   超时/连接失败/5xx/429 等可重试错误重试，4xx 不重试。
4. **熔断机制** — `CircuitBreaker`：连续失败达 `failure_threshold` 即 OPEN，冷却
   `cooldown_seconds` 后进入 HALF_OPEN 探测，成功自动恢复为 CLOSED。
5. **限流控制** — `RateLimiter` 令牌桶：`max_calls` 次 / `window_seconds`，超出等待
   最多 `max_wait_seconds` 后抛 `LLM_RATE_LIMITED`（HTTP 429），防止超配额。
6. **日志埋点** — `CallLogger`：在 调用开始 / 成功 / 错误 / 重试等待 / 失败 / 熔断状态变更
   处记录 request_id、model、host、耗时、响应摘要、异常；`api_key` 自动脱敏。

## 传输层与扩展性
- `OpenAITransport`（官方 `openai` SDK，优先）+ `UrllibTransport`（标准库零依赖回退）
  + `MockTransport`（mock 模式）。`build_transport()` 按配置自动选择。
- 未安装 `openai` 时自动回退 urllib，项目零依赖仍可运行（如需 SDK：`pip install openai`）。
- 新增传输只需实现 `BaseTransport.complete`，即插即用。

## 验证
- 冒烟测试 10/10 通过：mock 链路、重试+熔断开启、半开自动恢复、令牌桶限流、密钥脱敏。
- `app.py` 与 `llm_client.py` 均通过 `py_compile`。

## 使用
1. `cp .env.example .env`，填入 `LLM_API_KEY`（或临时设 `LLM_MOCK=true` 验证链路）。
2. `python app.py` → 访问 `http://localhost:8000/`，`POST /api/ai/parse` 即走韧性链路。
3. 调参：`llm_config.json` 中 retry / circuit_breaker / rate_limit / logging 段落。
