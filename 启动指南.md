## 1. 项目简介与架构概述

**智能课程表 Pro** 是一个「本地优先」的课程表 Web 应用：后端用 Python 标准库 `http.server` 实现，零外部框架；前端是单个 `smart-timetable-pro.html` 页面，无需构建步骤。

核心能力：

| 模块 | 说明 |
|------|------|
| 课程数据服务 | `GET /api/courses` 读取 `test_data.js` 返回课程 JSON，支持星期/周次/关键词过滤 |
| 用户配置系统 | 账号注册登录（`/api/auth/*`）+ 每用户的 API 地址/模型/Key/课程配置持久化（MySQL，Key 加密存储） |
| 统一日志系统 | 控制台彩色 + 文件按天滚动（gzip 压缩），含 `request_id` 链路追踪 |

**分层架构（前端 → 后端）：**

```
浏览器 (smart-timetable-pro.html)
        │  HTTP / REST
        ▼
app.py  (ThreadingHTTPServer 请求入口、路由、request_id 绑定)
   ├── llm_client.py   大模型调用（韧性：重试/熔断/限流）
   ├── auth.py         注册/登录/Session 签发
   ├── db.py           MySQL 连接池 + DAO（参数化查询防注入）
   ├── crypto.py       Fernet 加密(API_KEY) + PBKDF2 密码哈希 + Session 签名
   └── logger.py       统一日志（控制台 + 滚动文件 + 容错）
        │
        ▼
MySQL (可选；不装则用户配置模块优雅降级)
```

**优雅降级原则**：未配置 MySQL 或未装 `PyMySQL` 时，用户配置模块自动关闭（`user_config_enabled=false`），课程表与前端仍能正常运行；未配大模型 Key 时，仅 `/api/ai/parse` 返回 503，其余功能不受影响。

## 2. 运行环境要求

| 项目 | 要求 | 备注 |
|------|------|------|
| 操作系统 | Windows 10/11（本机 `win32`）；Linux / macOS 亦可 | `start_server.bat` 仅 Windows；其他系统用 `python app.py` |
| Python | **3.8+**（推荐 3.13） |  |
| pip | 随 Python 自带 | 用于安装可选依赖 |
| MySQL（可选） | 8.0+（字符集 utf8mb4） | **仅启用用户配置系统才需要**；不装也能跑主体 |
| 网络 | 本地回环即可；调用真实大模型需外网 | `/api/ai/parse` 需访问 LLM 服务地址 |
| 端口 | `8000` 未被占用 | 被占用可改 `app.py` 中 `PORT` |

> 依赖总览（按需安装，非全必需）：
> - `PyMySQL` + `cryptography`：**启用用户配置系统必需**
> - `openai`：真实调用大模型可选（不装走标准库回退）
> - 其余均为 Python 标准库，无需安装

## 5. 环境变量与配置文件配置说明

### 5.1 配置文件一览

| 文件 | 作用 | 是否必改 |
|------|------|----------|
| `.env.example` | 环境变量模板 | 复制为 `.env` 后按需填写 |
| `llm_config.json` | 大模型 base_url / model / 韧性参数 | 一般不用改（可用 `.env` 覆盖） |
| `logging_config.json` | 日志级别/路径/滚动策略 | 一般不用改（可用环境变量覆盖） |
| `schema.sql` | 数据库建表 | 启用 MySQL 时才执行 |

### 5.2 创建 .env

```bash
# Windows (cmd)
copy .env.example .env

# Linux / macOS
cp .env.example .env
```

### 5.3 关键变量说明

> ⚠️ **重要加载顺序说明**：`.env` 由 `llm_client` 在运行时（`load_config()`）自动加载，**主要用于 LLM 配置**。
> 而 `app.py` 在导入阶段就会读取 `DB_*` 检测 MySQL 连通性。因此 **`DB_*` 与 `TIMETABLE_SECRET_KEY` 最稳妥的做法是在启动前于终端导出为真实环境变量**（见 5.4）；`.env` 中的同名项在请求处理阶段也能生效。

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `LLM_API_KEY` | 空 | 大模型 Key；不填则 `/api/ai/parse` 返回 503 |
| `LLM_BASE_URL` | `https://api.siliconflow.cn/v1` | 大模型接口地址 |
| `LLM_MODEL` | `Qwen/Qwen2.5-7B-Instruct` | 模型名 |
| `LLM_TIMEOUT` | `30` | 请求超时（秒） |
| `LLM_MOCK` | `false` | `true` 时走 Mock，不调真实 API（便于链路测试） |
| `TIMETABLE_SECRET_KEY` | 空（开发用临时密钥） | **生产必填**，32+ 字符随机串，用于加密 API_KEY 与签名 Session |
| `TIMETABLE_COOKIE_SECURE` | `false` | 生产（HTTPS）设为 `true` |
| `DB_HOST` | `127.0.0.1` | MySQL 主机 |
| `DB_PORT` | `3306` | MySQL 端口 |
| `DB_USER` | `root` | MySQL 用户 |
| `DB_PASSWORD` | 空 | MySQL 密码 |
| `DB_NAME` | `timetable_pro` | 数据库名 |
| `DB_CHARSET` | `utf8mb4` | 字符集 |
| `APP_ENV` | `dev` | 日志环境：`dev` / `test` / `prod` |
| `LOG_LEVEL` | （按环境） | 覆盖日志级别：`DEBUG` / `INFO` / `WARN` / `ERROR` |
| `LOG_DIR` | `./logs` | 日志目录 |
| `LOG_CONSOLE` | `true` | 是否输出控制台 |
| `LOG_FILE` | `true` | 是否写文件 |

### 5.4 推荐的变量设置方式（最稳妥）

**Windows (cmd)：**

```bat
set DB_HOST=127.0.0.1
set DB_PORT=3306
set DB_USER=root
set DB_PASSWORD=你的密码
set DB_NAME=timetable_pro
set TIMETABLE_SECRET_KEY=请换成一个32位以上随机串
set LLM_API_KEY=你的LLMKey
```

**Windows (PowerShell)：**

```powershell
$env:DB_HOST="127.0.0.1"
$env:DB_PORT="3306"
$env:DB_USER="root"
$env:DB_PASSWORD="你的密码"
$env:DB_NAME="timetable_pro"
$env:TIMETABLE_SECRET_KEY="请换成一个32位以上随机串"
$env:LLM_API_KEY="你的LLMKey"
```

**Linux / macOS：**

```bash
export DB_HOST=127.0.0.1
export DB_PORT=3306
export DB_USER=root
export DB_PASSWORD=你的密码
export DB_NAME=timetable_pro
export TIMETABLE_SECRET_KEY="请换成一个32位以上随机串"
export LLM_API_KEY=你的LLMKey
```

生成随机密钥（任选其一）：

```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

> 提示：若仅本地体验、不启用 MySQL 用户配置系统，可**完全跳过数据库相关变量**；`TIMETABLE_SECRET_KEY` 留空时开发模式会用临时密钥（日志会有告警，本地不影响使用）。

## 6. 数据库初始化与迁移步骤

> 仅当**启用用户配置系统（账号 + 配置持久化）**时才需要。不执行本节，项目仍能正常提供课程表服务（用户配置模块自动降级）。

### 6.1 安装并启动 MySQL

- 下载安装 MySQL 8.0+，启动服务。
- 确认可用 `mysql` 客户端命令（加入 PATH）。

### 6.2 执行建库建表脚本

```bash
mysql -u root -p < schema.sql
```

执行成功会看到：

```
+------------------------------+
| message                      |
+------------------------------+
| 数据库 timetable_pro 初始化完成 |
+------------------------------+
```

该脚本会：

- 创建数据库 `timetable_pro`（utf8mb4）；
- 建表 `users`（账号，密码加盐哈希）、`user_configs`（配置，API_KEY 加密存储，与用户一对一）、`user_config_audit`（可选审计表）；
- 建立外键 `ON DELETE CASCADE` 与必要索引。

### 6.3 配置连接并验证

按第 5 节导出 `DB_*` 环境变量（确保与 `schema.sql` 中的库名/账号一致）。启动后访问：

```bash
curl http://127.0.0.1:8000/api/health
```

若返回中 `"user_config_enabled": true`，说明数据库已连上并启用。

### 6.4 迁移 / 变更说明

- 当前无 ORM / 迁移工具，表结构变更直接修改 `schema.sql` 后重新执行（或手写 `ALTER` 语句）。
- `schema.sql` 中 `DROP TABLE IF EXISTS` 会先删后建——**生产环境请勿直接重跑**，应手写增量迁移脚本。

## 8. 常见问题排查与故障诊断

| 现象 | 原因 | 解决方案 |
|------|------|----------|
| `Address already in use` / 端口被占用 | 8000 已被占用 | 查占用：`netstat -ano \| findstr :8000`（Linux `lsof -i:8000`），结束进程；或改 `app.py:61` 的 `PORT` 后重启 |
| `ModuleNotFoundError: No module named 'pymysql'` | 未装数据库驱动 | `pip install PyMySQL cryptography` |
| 启动后课程表正常，但用户配置用不了 | MySQL 未启动 / `DB_*` 未导出 / 未跑 `schema.sql` | 属**预期降级**；按第 5、6 节启用即可 |
| `[crypto] 警告：未设置 TIMETABLE_SECRET_KEY，使用临时密钥` | `.env` 未设主密钥 | 开发可忽略；生产务必 `python -c "import secrets;print(secrets.token_urlsafe(32))"` 生成并导出 |
| `/api/ai/parse` 返回 503 | 未配 `LLM_API_KEY` 且非 Mock | 在 `.env` 填 Key，或设 `LLM_MOCK=true` 做链路测试 |
| 双击 `start_server.bat` 一闪而过 | `app.py` 缺失或 Python 未找到 | bat 已内置检测；确认 `app.py` 与 bat 同目录；确认 Python 已加入 PATH |
| 前端能开但课程为空 | `test_data.js` 缺失或格式错误 | 检查根目录 `test_data.js` 存在且为合法 JS |
| `DB_*` 改了不生效 | `.env` 的 `DB_*` 在导入期未被读取 | 按 5.4 在**启动前于终端导出** `DB_*` 为真实环境变量 |
| 日志目录写入失败但不影响运行 | 日志目录无写权限 / 磁盘满 | 日志系统有容错（自动降级并告警），不影响主流程；检查 `LOG_DIR` 权限 |
| 控制台中文乱码 | 终端编码问题 | Windows 用 `chcp 65001`；或检查 IDE 终端编码为 UTF-8 |

**查看运行日志：** 项目根目录 `logs/app.log`（纯文本、字段对齐，无颜色码），`prod` 环境额外有 `logs/error.log`（仅 ERROR）。

## 9. 停止与清理操作说明

### 9.1 停止服务

- **bat 窗口**：`Ctrl+C`，会打印退出码并保持窗口（可查看末尾日志）。
- **命令行启动**：`Ctrl+C` 中断。
- **后台 / 忘记窗口**：

```bash
# Windows：找到占用 8000 的进程并结束
netstat -ano | findstr :8000
taskkill /PID <PID> /F

# Linux / macOS
lsof -ti:8000 | xargs kill -9
```

### 9.2 清理（按需）

| 清理项 | 命令 | 说明 |
|--------|------|------|
| 运行时日志 | `rmdir /s /q logs`（Win） / `rm -rf logs`（Linux） | 日志可随时删，重启自动重建 |
| Python 缓存 | `rmdir /s /q __pycache__` / `rm -rf __pycache__` | 不影响运行 |
| 虚拟环境 | 删除 `venv/` 目录 | 重装依赖即可恢复 |
| 数据库（谨慎） | `mysql -u root -p -e "DROP DATABASE timetable_pro;"` | **会清空所有用户配置**，仅测试环境操作 |

### 9.3 重新启动检查清单

1. 依赖已装（`pip install -r requirements.txt`）；
2. 需要用户配置系统时：MySQL 已启动、`schema.sql` 已执行、`DB_*` 已导出；
3. 需要大模型时：`LLM_API_KEY` 已设或 `LLM_MOCK=true`；
4. 端口 8000 空闲；
5. 运行 `python app.py`（或双击 `start_server.bat`）；
6. `curl http://127.0.0.1:8000/api/health` 返回 `{"ok":true,...}` 即成功。
