# 智能课程表 Pro — 运行与 AI 接入说明

本地优先的纯前端课程表应用，配套零依赖 Python 后端（`app.py`），提供课程数据接口与 AI 课余规划代理。

## 1. 本地运行

```bash
# 1) 安装依赖：无需任何三方包，只要 Python 3.8+
python app.py
# 2) 浏览器打开
#    http://localhost:8000/
```

- 课程表数据来自 `test_data.js`（由后端 `GET /api/courses` 透传）。

## 2. AI 空闲规划配置（.env）

在 `.env`（复制自 `.env.example`）中配置以下项即可启用「⚡ AI 生成复习建议」：

| 配置项 | 默认值 | 说明 |
|---|---|---|
| `TIMETABLE_AI_API_KEY` | 空 | AI 服务商 API Key。**仅存服务端**，绝不出现在前端源码或任何接口响应中。留空则端点自动降级为本地规则版。 |
| `TIMETABLE_AI_BASE_URL` | `https://api.openai.com/v1` | OpenAI 兼容接口地址，可替换为 DeepSeek / 通义 / 本地网关 等。 |
| `TIMETABLE_AI_TIMEOUT` | `8` | 单次请求超时（秒）。 |
| `TIMETABLE_AI_RATE_LIMIT` | `1` | 后端令牌桶限流：每秒最多请求数（单进程内生效）。 |
| `TIMETABLE_AI_MODEL` | `gpt-4o-mini` | 默认模型名。 |
| `TIMETABLE_CORS_ORIGIN` | `http://localhost:8000` | 允许跨域的前端来源（逗号分隔）。仅命中来源会收到 `Access-Control-Allow-Origin` + `Allow-Credentials`，非法来源不回显，避免 `*` + `credentials` 并存。 |

### 工作原理

前端在「空闲规划助手」弹窗点击 AI 按钮后，会组装上下文（日期、星期、单双周、空闲时间段、待办标签、规划类型、当日课程快照）并以 `POST /api/ai/plan` 发给**本机后端**；后端用标准库 `urllib.request` 将 `messages` 原样转发到 `{TIMETABLE_AI_BASE_URL}/chat/completions`，并回传 AI 文本。

## 3. AI 容错与降级

`/api/ai/plan` 对以下情况**统一降级**（HTTP 始终 200，前端据此回退到本地规则版规划）：

- 未配置 Key → `AI_DISABLED`
- 后端本地限流 → `AI_RATE_LIMITED`
- 请求超时 → `AI_TIMEOUT`
- 网络中断 → `AI_NETWORK`
- 上游 429 → `AI_UPSTREAM_429`
- 上游 4xx / 5xx → `AI_HTTP_4XX` / `AI_HTTP_5XX`
- 响应 JSON 解析失败 → `AI_JSON`
- 返回内容为空 → `AI_EMPTY`

> 说明：规则版 fallback 由**前端**负责（复用既有 `freeForDate` + `idleBlocks` + 平均分配逻辑），后端降级只回 `plan:null` + `reason`，内部 `reason` 仅入日志、不展示给终端用户。

## 4. 安全加固说明

- **CORS 收紧**：非法 `Origin` 不再收到 `Access-Control-Allow-Origin: *`，且 `Allow-Credentials` 仅对受信来源下发（见 `TIMETABLE_CORS_ORIGIN`）。
- **配置文件防下载**：`logging_config.json` 已移至 `config/` 目录，且静态服务白名单（`STATIC_EXTS`）已移除 `.json`，因此 `GET /config/logging_config.json` 不再可被下载。
- **密钥隔离**：AI Key 仅存服务端 `.env`，绝不出现在前端源码 / 响应 / 日志明文。

## 5. 前端单元测试

空闲规划相关的纯函数已抽取到 `timetable-core.js`（双环境导出），单测零依赖：

```bash
node --test tests/test_core.mjs
```

覆盖：`idleBlocks` / `deriveWeekType` / `validateCourseForm` / `computeInitialWeeks` / `softDeleteCore` / `undoDeleteCore` / `buildICSLines`（含相邻/覆盖/最小 20 分钟阈值、单双周 RRULE、非法表单、撤销、ICS 行拼接等边界）。

## 6. 已知限制

- AI 仅做文本建议，返回纯文本 / 宽松 Markdown；解析失败不致命，前端直接渲染原文。
- 后端为单进程代理，令牌桶限流为进程内语义；多进程部署需额外共享限流（如 Redis），不在本次范围。
- 当前为纯本地模式：无数据库、无用户账户/云端同步，所有课程表与配置均存储在浏览器本地（localStorage），课程数据由后端 `GET /api/courses` 透传 `test_data.js`。
