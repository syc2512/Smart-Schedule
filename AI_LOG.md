二、核心 Prompt
案例 1：空闲规划分段纯函数设计（N 标签 → N 时间段）
这是本次迭代中逻辑最复杂的纯算法需求，核心难点是在「时长均匀分配」与「绝不跨越课程间隙」两个约束间做平衡，同时保证纯函数可测试、双环境可用。
核心 Prompt：
plaintext
你是一名专注于零依赖前端架构的工程师，请为课程表项目设计空闲时间分段核心算法。
需求背景：用户框选当天的多段空闲时间，同时选择 N 个待办标签，需要将空闲总时长均匀分成 N 段，每段对应一个标签。
硬性约束：
1. 每个时间段必须完整落在同一个空闲块内，绝对不能跨越两段空闲之间的课程间隙
2. 每段时长至少 20 分钟，总时长不足时返回结构化错误信息
3. 纯函数实现，不依赖 DOM、全局状态，输入输出完全透明
4. 支持浏览器和 Node.js 双环境运行，可直接编写单元测试
输出要求：
1. 完整函数签名、入参出参数据结构定义
2. 详细算法伪代码，说明均分逻辑、跨块处理、极端边界兜底的实现细节
3. 至少 6 组单元测试用例，覆盖均分、块内拆分、时长不足、空输入、乱序归一、极端夹紧等场景
落地成果：
AI 输出了「块内顺序浇筑 + 整数整除均分 + 余数补给前序段 + 极端夹紧兜底」的完整算法方案，即 splitIdlePlan 纯函数。该方案完美满足 “不跨间隙” 的硬性约束，同时最大化保证时长均分；配套单测覆盖全部边界场景，落地后 node --test 全绿，前端「平均分配」与「AI 生成」两个场景均复用该内核，无逻辑漂移。
案例 2：大模型韧性调用层设计
后端接入大模型时，需要在「零强制三方依赖」的前提下，实现工业级调用稳定性，包含超时、重试、熔断、限流、脱敏日志等完整能力。
核心 Prompt：
plaintext
你是一名 Python 后端工程师，请为本地部署的课程表项目设计大模型调用客户端模块。
核心要求：
1. 零强制第三方依赖：默认用 Python 标准库 urllib 实现请求，可选兼容官方 openai SDK
2. 完整韧性能力：超时控制、指数退避+抖动重试、熔断器、令牌桶限流，所有异常均不崩溃
3. 安全要求：API 密钥仅从环境变量/.env 读取，日志中自动脱敏密钥
4. 可配置性：所有参数（超时时间、重试次数、熔断阈值、限流配额）均可通过配置文件调整

2. 核心类的关键方法与状态机逻辑
3. 统一返回格式与错误码定义
落地成果：
AI 输出了分层解耦的 llm_client.py 设计，包含传输层抽象、重试机制、熔断器状态机、令牌桶限流器、全链路日志埋点五大模块。落地后冒烟测试 10/10 通过，支持无 SDK 零依赖运行，异常场景均能平稳降级，完全符合项目轻量部署的定位。
三、Debug 经历
问题：空闲时间规划助手页面白屏报错
1. 现象描述
新增空闲规划助手页面后，点击首页入口跳转后页面完全空白，无 UI 渲染；终端无编译报错，其余静态页面访问正常，仅该功能页面失效。
2. 排查过程
按照标准排障路径，引导 AI 从外到内逐层定位：
第一步：捕获核心报错，锁定范围
首先打开浏览器开发者工具，捕获到控制台报错：Uncaught TypeError: Cannot read properties of undefined (reading 'push')，报错栈指向空闲规划页面的初始化逻辑。同步对照「路由配置 → 组件导入 → 依赖检查 → 入口语法 → 数据状态」5 个维度逐一校验，先排除明显低级错误。
第二步：排除路由与导入错误
先检查前端路由配置表，确认页面路径、组件名称拼写无误，无遗漏或拼写错误；再核对组件文件的相对路径与导出方式，确认文件存在且导出语句与导入引用匹配，排除路径与导入导出类问题。
第三步：定位根因：脚本加载顺序错误
进一步分析报错上下文，发现报错发生在页面初始化向全局状态 state.selfPlans 写入数据时。排查页面脚本加载顺序后发现：空闲规划页面的业务脚本被放在了 <head> 中提前加载，而全局状态对象 state 在页面底部才完成初始化，导致脚本执行时状态对象尚未定义。
第四步：修复与防御性加固
调整脚本加载顺序，将所有业务组件脚本移至全局状态与工具函数脚本之后加载
在页面初始化逻辑中增加防御性判断，状态未就绪时不执行业务初始化
沉淀脚本加载规范，明确「工具层 → 状态层 → 业务层」的加载顺序要求
3. 修复结果
调整后页面正常加载，空闲块框选、标签选择、平均分配、AI 生成等全流程功能运行正常；同时沉淀了原生多脚本项目的加载规范，避免后续新增模块再出现同类隐性问题。
4. 经验总结
原生无构建的前端项目中，脚本加载顺序是高频隐性 Bug 来源；排查页面白屏类问题时，遵循「先看报错 → 验路由 → 查导入 → 核对依赖与加载顺序」的路径，可高效缩小范围、定位根因。
