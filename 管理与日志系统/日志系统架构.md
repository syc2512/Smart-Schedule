# 统一日志系统 — 数据流

```
业务模块 → get_logger(name) → 多 Handler 分发 → 容错兜底
```

## 业务模块

- `app.py` · `db.py` · `auth.py` · `crypto.py` · `llm_client.py`

## logger.py — LoggingManager 单例

- 核心方法：`get_logger(name)` · `bind_context(request_id)` · 配置驱动 · 异常隔离
- Formatter 格式：`时间戳 | 级别 | 模块 | [request_id] | 消息`

## 多 Handler 分发

### 1. ConsoleHandler

- 实时彩色输出
- 级别配色：`DEBUG=青` `INFO=绿` `WARN=黄` `ERROR=红`
- 开发环境首选

### 2. TimedRotatingFile（按时间滚动）

- 按天滚动（`when=midnight`）
- 保留 N 天备份
- gzip 自动压缩
- 默认 30 份

### 3. RotatingFile（按大小滚动）

- 按大小滚动
- `maxBytes=10MB`
- `backupCount=10`
- 高频日志场景

## 容错兜底

> ⚠ 文件写入失败 → 回退到 stderr + 内存缓冲，**绝不抛异常影响主业务**

## 数据流示意

```
业务模块
   │
   ▼
logger.py (LoggingManager 单例, get_logger / bind_context)
   │
   ├──────────────┬──────────────────────┬──────────────────┐
   ▼              ▼                      ▼                  ▼
ConsoleHandler  TimedRotatingFile    RotatingFile      (异常隔离)
(实时彩色)      (按天滚动, gzip)     (按大小, 10MB)
   │
   ▼
容错兜底：写入失败 → stderr + 内存缓冲
```
