# 用户配置数据管理系统

> 为智能课程表 Pro 增加基于 MySQL 的用户配置持久化能力。
> 用户首次登录填写 API 地址 / 模型名 / API Key / 学期信息等配置，系统加密存储到数据库；
> 后续登录自动从数据库读取并下发前端，无需重复填写。

---

## 一、架构概览

```
┌────────────────────────────────────────────────────────────────┐
│  前端  smart-timetable-pro.html                                 │
│  - 登录/注册抽屉 · 账户菜单 · 设置面板（自动加载/同步远端配置）  │
│  - AuthAdapter 模块封装所有 REST 调用                           │
└────────────────────────────┬───────────────────────────────────┘
                             │ fetch (Cookie 携带 Session)
┌────────────────────────────┴───────────────────────────────────┐
│  后端  app.py  (http.server + 路由分发)                          │
│  /api/auth/register | login | logout | me                       │
│  /api/user/config (GET/POST)  ·  /api/user/courses (GET/POST)   │
│  /api/health  ·  /api/courses  ·  /api/ai/parse  (原有保留)     │
└────────────────────────────┬───────────────────────────────────┘
                             │
        ┌────────────────────┼────────────────────┐
        ▼                    ▼                    ▼
   auth.py              crypto.py              db.py
   (鉴权服务)           (加密服务)             (DAO + 连接池)
   PBKDF2 密码哈希      Fernet API_KEY 加密    PyMySQL 驱动
   HMAC Session 票据    bcrypt 风格 salt       参数化查询防注入
                             │
                             ▼
                        MySQL 8.0
```

**核心特性**

| 维度 | 实现 |
|------|------|
| 用户账号 | `users` 表，密码 PBKDF2-HMAC-SHA256 + 16 字节 salt，迭代 200,000 次 |
| 配置存储 | `user_configs` 表，与 `users` 一对一（外键 + ON DELETE CASCADE） |
| 敏感字段加密 | `api_key_encrypted` 字段使用 Fernet (AES-128-CBC + HMAC-SHA256) 加密 |
| Session 鉴权 | 签名 Cookie（HMAC-SHA256），HttpOnly + SameSite=Strict，7 天有效期 |
| API 风格 | REST，JSON，统一错误码 `{error, code}` |
| 优雅降级 | MySQL 不可用时，所有用户配置 API 返回 503，前端自动回退到 localStorage |

## 二、数据库表结构

详见 `schema.sql`，核心两张表：

### users — 用户账号

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT UNSIGNED PK | 自增主键 |
| username | VARCHAR(64) UNIQUE | 登录用户名（3-32 位字母数字下划线） |
| password_hash | VARCHAR(255) | PBKDF2 哈希后的密码（base64） |
| salt | VARCHAR(64) | 用户级随机盐（base64） |
| display_name | VARCHAR(128) | 显示名（可选） |
| is_active | TINYINT(1) | 账号是否启用 |
| last_login_at | DATETIME | 最近登录时间 |
| created_at / updated_at | DATETIME | 时间戳 |

### user_configs — 用户配置（一对一）

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | BIGINT UNSIGNED PK + FK | 关联 users.id |
| api_url | VARCHAR(512) | 大模型 API 地址 |
| model_name | VARCHAR(128) | 大模型名 |
| api_key_encrypted | TEXT | Fernet 加密后的 API_KEY |
| courses_json | JSON | 课程数据数组 |
| semester_start | DATE | 学期起始日 |
| total_weeks | INT | 学期总周数 |
| extra_settings | JSON | 扩展配置（前端自定义） |
| created_at / updated_at | DATETIME | 时间戳 |

## 三、REST API 规范

所有 API 返回 JSON，统一错误格式：`{"error": "...", "code": "..."}`

### 1. 注册

```
POST /api/auth/register
Content-Type: application/json

{
  "username": "alice",
  "password": "Alice2025!",
  "display_name": "Alice"        // 可选
}

// 201 Created
{
  "user": {"id": 1, "username": "alice", "display_name": "Alice"},
  "expires_in": 604800
}
// 同时 Set-Cookie: timetable_session=...; HttpOnly; SameSite=Strict
```

错误码：`AUTH_INVALID_USERNAME` (400) · `AUTH_WEAK_PASSWORD` (400) · `AUTH_USERNAME_TAKEN` (409)

### 2. 登录

```
POST /api/auth/login
Content-Type: application/json

{"username": "alice", "password": "Alice2025!"}

// 200 OK
{"user": {...}, "expires_in": 604800}
// Set-Cookie: timetable_session=...
```

错误码：`AUTH_INVALID_CREDENTIALS` (401) · `AUTH_ACCOUNT_DISABLED` (401)

### 3. 登出

```
POST /api/auth/logout
// Set-Cookie: timetable_session=; Max-Age=0
{"ok": true}
```

### 4. 当前用户信息

```
GET /api/auth/me
Cookie: timetable_session=...

// 200 OK
{
  "user": {
    "id": 1, "username": "alice", "display_name": "Alice",
    "is_active": true, "has_config": true, "has_api_key": true,
    "last_login_at": "...", "created_at": "..."
  }
}
// 未登录: 401 {"error":"未登录或 Session 已过期","code":"AUTH_REQUIRED"}
```

### 5. 获取用户配置

```
GET /api/user/config
Cookie: timetable_session=...

// 已配置过 (200)
{
  "config": {
    "api_url": "https://api.siliconflow.cn/v1",
    "model_name": "Qwen/Qwen2.5-7B-Instruct",
    "api_key": "",                  // 永不下发明文
    "api_key_masked": "sk-x...abcd",// 掩码形式
    "has_api_key": true,
    "courses": [...],
    "semester_start": "2025-09-01",
    "total_weeks": 18,
    "extra_settings": {},
    "updated_at": "..."
  },
  "first_login": false
}

// 首次登录 (200)
{"config": null, "first_login": true, "message": "首次登录，请填写配置"}
```

### 6. 保存 / 更新用户配置

```
POST /api/user/config
Cookie: timetable_session=...
Content-Type: application/json

{
  "api_url": "https://api.siliconflow.cn/v1",
  "model_name": "Qwen/Qwen2.5-7B-Instruct",
  "api_key": "sk-new-key-here",     // 空字符串或不传 = 保留原值
  "semester_start": "2025-09-01",
  "total_weeks": 18
}

// 200 OK
{"ok": true, "config": {...}}     // 返回更新后的配置
```

字段白名单：`api_url, model_name, api_key, semester_start, total_weeks, extra_settings`
未提供的字段保留原值（部分更新语义）。

### 7. 获取 / 保存课程数据

```
GET  /api/user/courses  →  {"courses": [...]}
POST /api/user/courses
     Body: {"courses": [...]}
     →  {"ok": true, "count": 25}
```

课程上限 500 条，超过返回 400。

### 8. 健康检查

```
GET /api/health
{"ok": true, "user_config_enabled": true, "version": "1.1.0"}
```

## 四、部署步骤

### 前置条件

- Python 3.10+
- MySQL 8.0+（或 MariaDB 10.5+）

### 步骤

1. **安装依赖**

   ```bash
   pip install -r requirements.txt
   ```

2. **配置环境变量**

   ```bash
   cp .env.example .env
   # 编辑 .env，填入 MySQL 密码、生成 TIMETABLE_SECRET_KEY
   ```

   生成密钥：
   ```bash
   python -c "import secrets; print(secrets.token_urlsafe(32))"
   ```

3. **初始化数据库**

   ```bash
   mysql -u root -p < schema.sql
   ```

   这会创建 `timetable_pro` 数据库及 `users` / `user_configs` / `user_config_audit` 表。

4. **启动服务**

   ```bash
   python app.py
   # 或双击 start_server.bat (Windows)
   ```

   启动时会打印：
   ```
   用户配置: 已启用 (MySQL 已连接)
   ```

   表示模块已就绪。

5. **访问应用**

   浏览器打开 `http://localhost:8000/`，点击右上角用户图标注册账号。

## 五、安全设计

| 威胁 | 防护措施 |
|------|---------|
| **密码泄露** | PBKDF2-HMAC-SHA256 + 16 字节随机 salt，迭代 200,000 次；后端永不存储明文 |
| **API_KEY 泄露** | Fernet 对称加密（AES-128-CBC + HMAC-SHA256），密钥从环境变量读取；GET 接口不返回明文，仅返回掩码 |
| **Session 伪造** | HMAC-SHA256 签名票据，包含过期时间；任何字段被修改签名校验失败 |
| **Session 劫持** | Cookie 设置 HttpOnly + SameSite=Strict；生产环境 HTTPS 加 `Secure` 标志 |
| **SQL 注入** | 所有查询使用参数化绑定（`%s` 占位符），DAO 层强制类型校验 |
| **CSRF** | SameSite=Strict Cookie；写操作仅接受 POST + JSON body |
| **暴力破解** | 用户名/密码错误返回相同错误码（不区分"用户不存在"与"密码错误"）；可在前端加限流 |
| **时序攻击** | 密码校验使用 `hmac.compare_digest` 恒定时间比较 |
| **密钥泄露** | 主密钥仅从环境变量 `TIMETABLE_SECRET_KEY` 读取，绝不写入代码或配置文件 |

## 七、降级机制

当 MySQL 不可用时（未安装驱动、数据库未启动、网络不通等）：

- 启动时 `check_connection()` 返回 False，`_USER_CONFIG_ENABLED = False`
- 所有 `/api/auth/*` 和 `/api/user/*` 端点返回 503 `USER_CONFIG_DISABLED`
- 前端 `AuthAdapter.health()` 探测到 `user_config_enabled: false` 后：
  - 不弹登录界面
  - 沿用原有 localStorage + IndexedDB 本地存储模式
  - 课程表所有原有功能（增删改查、AI 解析、导出）继续可用

这意味着：**升级到新版后，原项目零依赖运行的特性不受影响**，仅在配置好 MySQL 时才启用用户配置管理。

## 八、开发与测试

### 单元自检

每个模块都带 `__main__` 自检：

```bash
python crypto.py    # Fernet 加密 + PBKDF2 + Session 票据
python auth.py      # 用户名/密码正则校验
python db.py        # MySQL 驱动加载 + 连通性
```

### 端到端测试（curl）

```bash
# 注册
curl -i -X POST http://localhost:8000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"alice","password":"Alice2025!"}' \
  -c cookies.txt

# 获取当前用户
curl -i http://localhost:8000/api/auth/me -b cookies.txt

# 保存配置
curl -i -X POST http://localhost:8000/api/user/config \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{"api_url":"https://api.siliconflow.cn/v1","model_name":"Qwen/Qwen2.5-7B-Instruct","api_key":"sk-test-123","semester_start":"2025-09-01","total_weeks":18}'

curl -i http://localhost:8000/api/user/config -b cookies.txt
```
