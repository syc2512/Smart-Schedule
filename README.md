# timetble
智能课程表 Pro
## 一、项目基本信息
⽬标：打造⼀个不仅好看，⽽且能够聪明地处理各种复杂教学⽇程的课表助⼿。
-基础功能（及格线）：
拥有美观的周课表、⽇课表视图，⽀持⼿动添加、修改、删除课程。
课表需清晰展示单双周、上课地点、任课⽼师，⽀持不同课程颜⾊⾼亮。
-进阶功能（Vibe Coding 核⼼，⾄少实现⼀项）：
AI 驱动⼀键导⼊：⽤户可以直接复制学校教务系统混乱的⽂本，或者上传⼀张教务处排课表格的截图。系统对接 AI 接⼝（或前端调⽤免费⼤模型接⼝），⾃动将⾮结构化数据解析为标准课表数据导⼊。
空闲⾃习规划：系统⾃动计算⽤户的“⽆课空闲时间段”，并提供⼀个“AI 课余规划助⼿”，⼀键为⽤户⽣成今⽇⾃习、考公或考研的复习建议。
## 二、项目相关功能
- 周课表、日课表视图
- 课程的手动增 / 改 / 删
- 单双周、上课地点、任课教师的清晰展示
- 课程颜色高亮
- AI 一键导入（文本粘贴 / 截图上传 → 标准课表）
- 空闲自习规划 + AI 课余规划助手
- 导出 `.ics` 日历文件

## 三、项目简要介绍

### （一）项目背景
高校学生普遍面临以下痛点：
1. **教务系统数据割裂**：教务系统导出的课表常为混乱的纯文本、PDF 或截图，无法直接复用。
2. **多端不同步**：Web 课表与手机系统日历割裂，学生需手动在 iPhone/Android 日历中再录入一次。
3. **空闲时间浪费**：学生难以快速识别"无课时段"，缺乏针对考研/考公/自习的结构化复习建议。
4. **复杂教学规则**：单双周（奇偶周）交替、调停课、多校区地点、多位任课教师等规则在传统课表 App 中支持薄弱。
### （二）使用场景
- **场景 A（开学导入）**：大三学生小陈从教务系统复制了一坨乱码文本，粘贴进 App，AI 自动解析出 12 门课并渲染周课表，全程 < 1 分钟。
- **场景 B（日常查看）**：考研生小林每天早上面向"日课表"查看今天几节课，剩余无课时段被标记为"可复习"，点击"AI 规划"拿到今日考研复习清单。
- **场景 C（多端同步）**：同步用户小王将整学期课表导出为 `.ics`，导入 iPhone 日历，上课前自动收到系统提醒。
- **场景 D（异常处理）**：懒人用户上传了一张模糊的排课截图，AI 解析失败；系统给出友好提示，允许用户手动修正或重新上传，且**不崩溃、不卡死**。
### （三）用户角色
| 角色 | 简称 | 核心诉求 | 典型目标 |
| --- | --- | --- | --- |
| 普通在校大学生 | 学生 | 快速建表、随时查看、不错过课 | 美观清晰地掌握每周课程 |
| 考研党 | 考研生 | 在有限无课时段高效复习 | 生成考研科目轮动复习计划 |
| 考公党 | 考公生 | 行测/申论模块化刷题安排 | 生成考公备考节奏建议 |
| 多设备重度用户 | 同步用户 | 课表进手机日历、不错漏 | 一键导出 .ics 到系统日历 |
| 非技术/手残用户 | 懒人用户 | 不手动录入，截图即导入 | AI 解析截图自动建课 |

### （四）业务流程说明
### 4.1 手动建课流程
1. 用户进入"添加课程" → 填写表单（F-03）；
2. 前端实时校验字段合法性（FR-S1）；
3. 检测时间冲突 → 若冲突则弹预警（不阻断）；
4. 提交 → 写入本地存储 → 周/日视图即时刷新（F-01/F-02/F-07）。

### 4.2 AI 一键导入流程（F-08）
1. 用户选择"文本粘贴"或"上传截图"；
2. 系统做输入校验（长度/类型/大小，FR-S1）；
3. 调用 AI 接口（try/catch + 超时）→ 见踩坑点①；
4. 成功：解析为标准课程数组 → 预览（标红可疑字段）→ 用户确认；
5. 失败：捕获异常 → 友好提示 + 引导手动录入，**不崩溃**；
6. 确认后：字段二次校验 → 写入本地 → 渲染。

### 4.3 空闲自习规划流程（F-09）
1. 系统读取课表，按当前周次（单/双）过滤；
2. 计算 07:00–22:00 内无课时段，剔除 < 20 分钟碎片；
3. 读取用户目标模式（自习/考研/考公）；
4. 调用 AI 生成复习计划（带降级兜底）；
5. 展示空闲块 + 计划；支持"加入提醒/导出 .ics"。

### 4.4 日历导出流程（F-10）
1. 用户选择导出范围（整学期/本周/所选周）；
2. 系统按课程生成 VEVENT（含单双周拆分规则）；
3. 组装标准 `.ics` 字符串 → 提供下载；
4. 用户导入手机系统日历 → 验证事件显示与提醒正常。
