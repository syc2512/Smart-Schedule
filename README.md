# timetble
智能课程表 Pro
## 一、项目基本信息
⽬标：打造⼀个不仅好看，⽽且能够聪明地处理各种复杂教学⽇程的课表助⼿。
-基础功能（及格线）：
拥有美观的周课表、⽇课表视图，⽀持⼿动添加、修改、删除课程。
课表需清晰展示单双周、上课地点、任课⽼师，⽀持不同课程颜⾊⾼亮。
-进阶功能（Vibe Coding 核⼼，⾄少实现⼀项）：
AI 驱动⼀键导⼊：⽤户可以直接复制学校教务系统混乱的⽂本，或者上传⼀张教务处排课表格的截图。系统对接 AI 接⼝（或前端调⽤免费⼤模型接⼝），⾃动将⾮结构化数据解析为标准课表数据导⼊。
空闲⾃习规划：系统⾃动计算⽤户的“⽆课空闲时间段”，并提供⼀个“AI 课余规划助⼿”，⼀键为⽤户⽣成今⽇⾃习、考公或考研的复习建议。
## 二、项目相关功能
**包含（In Scope）：**
- 周课表、日课表视图
- 课程的手动增 / 改 / 删
- 单双周、上课地点、任课教师的清晰展示
- 课程颜色高亮
- AI 一键导入（文本粘贴 / 截图上传 → 标准课表）
- 空闲自习规划 + AI 课余规划助手
- 导出 `.ics` 日历文件

**不包含（Out of Scope）：**
- 实时对接教务系统后台（爬虫/API 抓课）【假设：仅支持用户导出/截图后导入】
- 在线选课、退课、成绩查询
- 用户社交、课表分享社区
- 支付、广告、账号付费体系（v1）
- 跨设备云同步（v1 仅本地存储）【假设】

## 三、项目简要介绍

### （一）项目背景
高校学生普遍面临以下痛点：
1. **教务系统数据割裂**：教务系统导出的课表常为混乱的纯文本、PDF 或截图，无法直接复用。
2. **多端不同步**：Web 课表与手机系统日历割裂，学生需手动在 iPhone/Android 日历中再录入一次。
3. **空闲时间浪费**：学生难以快速识别"无课时段"，缺乏针对考研/考公/自习的结构化复习建议。
4. **复杂教学规则**：单双周（奇偶周）交替、调停课、多校区地点、多位任课教师等规则在传统课表 App 中支持薄弱。
### （二）使用场景
- **场景 A（开学导入）**：大三学生小陈从教务系统复制了一坨乱码文本，粘贴进 App，AI 自动解析出 12 门课并渲染周课表，全程 < 1 分钟。
- **场景 B（日常查看）**：考研生小林每天早上面向"日课表"查看今天几节课，剩余无课时段被标记为"可复习"，点击"AI 规划"拿到今日考研复习清单。
- **场景 C（多端同步）**：同步用户小王将整学期课表导出为 `.ics`，导入 iPhone 日历，上课前自动收到系统提醒。
- **场景 D（异常处理）**：懒人用户上传了一张模糊的排课截图，AI 解析失败；系统给出友好提示，允许用户手动修正或重新上传，且**不崩溃、不卡死**。
### （三）用户角色
| 角色 | 简称 | 核心诉求 | 典型目标 |
| --- | --- | --- | --- |
| 普通在校大学生 | 学生 | 快速建表、随时查看、不错过课 | 美观清晰地掌握每周课程 |
| 考研党 | 考研生 | 在有限无课时段高效复习 | 生成考研科目轮动复习计划 |
| 考公党 | 考公生 | 行测/申论模块化刷题安排 | 生成考公备考节奏建议 |
| 多设备重度用户 | 同步用户 | 课表进手机日历、不错漏 | 一键导出 .ics 到系统日历 |
| 非技术/手残用户 | 懒人用户 | 不手动录入，截图即导入 | AI 解析截图自动建课 |
### （四）功能需求清单
### 4.1 P0 — 基础功能（及格线）

#### F-01 周课表视图 ｜ P0
- **功能描述**：以周一至周日（或自定义周起始日）为横轴、节次/时间为纵轴展示整周课程矩阵。
- **输入**：用户已有的课程数据集（本地存储）。
- **输出**：7×N 网格化课表，单元格显示课程名、地点、教师、单双周标记。
- **业务规则**：
  - 单元格按课程颜色高亮（见 F-07）；
  - 单双周课程需以角标（如"单"/"双"）或透明度区分；
  - 当前日期列高亮；
  - 无课单元格显示空白或"空闲"占位。

#### F-02 日课表视图 ｜ P0
- **功能描述**：聚焦单日，按时间轴纵向列出当日课程与空闲块。
- **输入**：选定日期 + 课程数据集。
- **输出**：当日时间轴列表（含课程与空闲时段标注）。
- **业务规则**：
  - 默认打开"今天"；
  - 自动折叠/标注无课长时段为"空闲 X 小时"；
  - 与周视图数据一致、可一键切换。

#### F-03 手动添加课程 ｜ P0
- **功能描述**：提供表单让用户手动新增一门课程。
- **输入**：课程名、星期、起始/结束节次或起止时间、地点、教师、单双周类型、颜色（可自动分配）。
- **输出**：新课程写入本地存储并即时渲染。
- **业务规则**：
  - 课程名必填；星期、时间必填；
  - 时间不得与同周次已有课程重叠（重叠时**预警**而非静默覆盖）【假设：预警而非禁止，便于调停课】；
  - 非法/空字段提交时前端即时校验并提示，不写入脏数据。

#### F-04 修改课程 ｜ P0
- **功能描述**：点击课表单元格进入编辑，修改课程任意字段。
- **输入**：课程 ID + 修改后的字段值。
- **输出**：更新后数据 + 重新渲染。
- **业务规则**：与 F-03 共用校验逻辑；修改后保留课程唯一 ID（不新建记录）。

#### F-05 删除课程 ｜ P0
- **功能描述**：删除单门课程（支持单条删除与"按名称批量删除"）。
- **输入**：课程 ID 或课程名。
- **输出**：数据移除 + 视图更新。
- **业务规则**：
  - 删除前二次确认弹窗；
  - 批量删除按名称匹配，提示受影响条数；
  - 删除为软删除（标记 `deleted`，可撤销 30 秒）【假设：降低误删损失】。

#### F-06 课程信息展示（单双周 / 地点 / 教师）｜ P0
- **功能描述**：在课表单元格与详情中完整展示单双周、地点、教师。
- **输入**：课程实体字段。
- **输出**：可视化展示。
- **业务规则**：
  - 单双周类型取值：`all`(每周)/`odd`(单周)/`even`(双周)，互斥；
  - 当前周次（单/双）由"学期起始日 + 周偏移"推算并随周切换；
  - 地点/教师为空时显示"—"而非报错。

#### F-07 课程颜色高亮 ｜ P0
- **功能描述**：为不同课程分配可区分的颜色，提升辨识度。
- **输入**：课程创建时自动分配或用户手动指定调色板索引。
- **输出**：单元格背景/边框着色。
- **业务规则**：
  - 默认从 12 色无障碍调色板循环分配；
  - 同名课程自动同色；
  - 提供手动改色入口；
  - 颜色对比度需满足 WCAG AA（文字可读）。

### 4.2 P1 — 进阶功能（Vibe Coding 核心，至少实现一项）

#### F-08 AI 一键导入（文本 / 截图解析）｜ P1
- **功能描述**：用户粘贴教务系统混乱文本，或上传排课截图；系统调用 AI 接口将非结构化数据解析为标准课程数组并预览确认后导入。
- **输入**：
  - 文本模式：用户粘贴的纯文本（≤ 8000 字符）【假设：上限防滥用】；
  - 图片模式：用户上传的截图（jpg/png/webp，≤ 10MB）【假设】；
  - 用户自有/系统配置的大模型 API（OpenAI 兼容协议）。
- **输出**：结构化课程数组（JSON），经用户确认后写入本地存储并渲染；展示"解析置信度"与可疑字段标记。
- **业务规则**：
  - **AI 容错捕获（踩坑点①）**：任何 AI 调用必须用 try/catch 包裹；超时（默认 15s）、网络错误、API 返回非预期格式、配额耗尽等均需兜底——返回友好错误并引导用户改用手动录入，**绝不直接抛出未捕获异常导致白屏**；
  - 解析结果先"预览→用户确认"再写入，不直接覆盖现有课表；
  - 对 AI 返回的每条课程执行与 F-03 相同的字段校验，非法字段标红提示；
  - 提供"重新解析 / 手动修正 / 放弃"三选项；
  - API Key 仅存于用户本地，明文不落日志、不上报【假设】。

#### F-09 空闲自习规划 + AI 课余规划助手 ｜ P1
- **功能描述**：系统自动计算当日/本周"无课空闲时间段"，并由 AI 结合用户目标（自习/考研/考公）生成可执行复习建议。
- **输入**：
  - 当前课表数据 + 目标模式（自习 / 考研 / 考公，用户可切换）；
  - 可选的每日可投入时长偏好【假设】。
  - AI 接口（同 F-08 通道）。
- **输出**：
  - 空闲时段列表（开始/结束/时长/星期）；
  - AI 生成的复习计划（按空闲块分配任务，含科目、时长、建议）。
- **业务规则**：
  - 空闲时段 = 当日 07:00–22:00 减去已排课程（按单双周过滤当前周）；
  - 单段空闲 < 20 分钟不纳入规划（碎片化）【假设】；
  - AI 规划失败时使用**内置规则兜底**（按目标模式套用预设模板，如考研=政治/英语/专业课轮动），保证"无 AI 也能出计划"；
  - 规划结果可一键"加入 .ics 提醒"或导出。

### 4.3 P2 — 亮点功能（多端联动）

#### F-10 日历多端联动（.ics 导出）｜ P2
- **功能描述**：将课表生成为符合 RFC 5545 国际标准的 `.ics` 文件，用户下载后导入 iPhone/Android 系统日历。
- **输入**：课程数据集 + 学期起止日 + 当前周次类型。
- **输出**：标准 `.ics` 文本文件（含 VEVENT，支持单双周 RRULE 或拆分为独立事件）。
- **业务规则**：
  - 每个课程生成对应 VEVENT，含 SUMMARY(课程名)、LOCATION(地点)、DESCRIPTION(教师)、DTSTART/DTEND；
  - 单双周通过 RRULE 的 INTERVAL=2 + 偏移，或拆分为"单周事件/双周事件"两种方式之一【假设：采用拆分方式兼容性最佳】；
  - 文件需通过 iOS/Android 实测导入验证（见验收标准）；
  - 导出前提供"选择导出范围（整学期/本周/所选周）"选项。

### 4.4 安全与健壮性专项

> 以下作为**横切非功能 + 功能级**硬性要求，贯穿所有功能。

- **FR-S1 非法输入防护**：所有用户输入（课程名、时间、教师、粘贴文本、上传文件）必须经过边界校验与无害化处理；超长截断、特殊字符转义、文件类型/大小校验；任何异常输入都不得导致渲染崩溃或 XSS。
- **FR-S2 AI 服务降级**：AI 接口不可用时，F-08 / F-09 必须降级到手动录入 / 内置规则模板，并以 Toast/横幅告知用户"AI 暂不可用，已切换为手动/本地模式"。
- **FR-S3 全局异常兜底**：前端设置全局 error boundary，任何未预期错误显示"出错了，可重试"页面而非白屏。

### （五）业务流程说明
### 5.1 手动建课流程
1. 用户进入"添加课程" → 填写表单（F-03）；
2. 前端实时校验字段合法性（FR-S1）；
3. 检测时间冲突 → 若冲突则弹预警（不阻断）；
4. 提交 → 写入本地存储 → 周/日视图即时刷新（F-01/F-02/F-07）。

### 5.2 AI 一键导入流程（F-08）
1. 用户选择"文本粘贴"或"上传截图"；
2. 系统做输入校验（长度/类型/大小，FR-S1）；
3. 调用 AI 接口（try/catch + 超时）→ 见踩坑点①；
4. 成功：解析为标准课程数组 → 预览（标红可疑字段）→ 用户确认；
5. 失败：捕获异常 → 友好提示 + 引导手动录入，**不崩溃**；
6. 确认后：字段二次校验 → 写入本地 → 渲染。

### 5.3 空闲自习规划流程（F-09）
1. 系统读取课表，按当前周次（单/双）过滤；
2. 计算 07:00–22:00 内无课时段，剔除 < 20 分钟碎片；
3. 读取用户目标模式（自习/考研/考公）；
4. 调用 AI 生成复习计划（带降级兜底）；
5. 展示空闲块 + 计划；支持"加入提醒/导出 .ics"。

### 5.4 日历导出流程（F-10）
1. 用户选择导出范围（整学期/本周/所选周）；
2. 系统按课程生成 VEVENT（含单双周拆分规则）；
3. 组装标准 `.ics` 字符串 → 提供下载；
4. 用户导入手机系统日历 → 验证事件显示与提醒正常。
